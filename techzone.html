Case Study

TechZone

A responsive web app for tracking PC component prices, exploring build guides, and managing a personalized watchlist with clear, accessible data presentation.

Role

Front-End Developer / UX Contributor

Stack

HTML, CSS, JavaScript, Bootstrap, APIs

Team

4-person team project

techzone

Overview

TechZone was built to help users compare PC parts and make better purchasing decisions. The product combines live pricing, category browsing, and personalized tracking tools in a single, responsive interface.

My Contributions & Learnings

  • Built responsive comparison interfaces.I implemented the front-end layout across screen sizes, which made pricing and product details easier to browse on desktop and mobile.
  • Improved content structure and readability.I organized listings, hierarchy, and interface spacing so users could scan categories and compare components with less effort.
  • Refined UX details around data-heavy views.I adjusted navigation, presentation patterns, and visual rhythm to make live pricing information feel clearer instead of overwhelming.
  • Learned how to design around live data.The project sharpened how I translate API-driven content into interfaces that stay usable, legible, and consistent.

Team Contributions

  • Scoped the product around shopper workflows.The team centered the experience on price tracking, build exploration, and watchlist management so the app solved the most common comparison tasks.
  • Integrated external data sources.We connected third-party APIs and shaped the interface around live pricing, which made the product more useful than a static catalog.
  • Tested the experience across breakpoints.The team reviewed responsive behavior together, which improved consistency for users comparing parts on different devices.

Outcome

Comparison became faster

Users could view prices, browse categories, and manage saved items in one place instead of jumping between multiple sites.

Dense data felt more approachable

Cleaner structure and clearer hierarchy made hardware shopping easier to scan, especially on smaller screens.

Key Features

Feature 01

Real-time pricing

External API data keeps component pricing current so users can make more informed purchase decisions.

Feature 02

Build guide browsing

Structured categories make it easier to explore hardware options and understand different build paths.

Feature 03

Personalized watchlist

Users can save products over time, giving them a simpler way to track parts they are considering.

Feature 04

Responsive browsing

The layout stays usable across devices so users can compare components without losing clarity on mobile.
